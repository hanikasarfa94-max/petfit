可以。按现在这套 IA，UI 物料不能只按“页面”切，要按 **角色资产、对象资产、动作资产、数据资产、状态资产、场景资产** 来拆。这样给设计/前端/Codex 都更清楚。

下面是完整拆解。

# PetFit UI 物料清单拆解

这是最高优先级。所有页面都依赖统一角色。

### 1.1 基础角色形象

需要一套统一三视图或基础形象：

```text
小布丁仓鼠
- 奶油金色身体
- 白色腹部
- 黑豆眼
- 粉色耳朵
- 粉色小爪
- 头顶草莓
- 蓝色斜挎小包
- 圆润治愈风格
```

### 1.2 主状态动作

| ------- | ----------- | ------- |

# 2. 三个核心对象物料

这是产品结构的核心：饭碗、水瓶、靠垫。

## 2.1 饭碗 / 餐盘

建议不要做“宠物碗感太强”，而是更像：

```text
温柔圆形餐盘 / 浅黄色托盘 / 小布丁餐盘
```

需要状态：

| 状态     | 说明          |
| ------ | ----------- |
| 空盘     | 今日未记录       |
| 少量食物   | 1–2 次记录     |
| 正常食物   | 3–5 次记录     |
| 丰富食物   | 6 次以上       |
| 过量堆叠   | 8 次以上，显示 +N |
| 选中高亮   | 当前日期被选中     |
| 保存进入动画 | 食物贴纸飞入餐盘    |

### 饭碗页核心规则

饭碗不是单个固定图，它要支持贴纸堆叠：

```text
餐盘底图 + 食物贴纸层 + 计数气泡 + 小布丁装饰
```

所以前端最好用分层结构，不要把食物全部画死在一张图里。

---

## 2.2 水瓶

水瓶是第二核心对象。建议使用半透明水瓶，能表达水位。

需要状态：

| 状态     | 说明        |
| ------ | --------- |
| 空瓶     | 今日无饮水     |
| 低水位    | 少量饮水      |
| 中水位    | 正常饮水      |
| 高水位    | 接近目标      |
| 满水位    | 达标        |
| 有饮料贴纸  | 拍到饮料      |
| 偏甜标签   | 奶茶、可乐、果汁等 |
| 水位上涨动画 | 快捷喝水后     |

### 水瓶内部物料

```text
水位层
水波纹
水滴小图标
饮料贴纸
偏甜小标签
瓶身小布丁贴纸
```

---

## 2.3 靠垫

靠垫用于休息记录。

需要状态：

| 状态     | 说明     |
| ------ | ------ |
| 空靠垫    | 今日未休息  |
| 轻度使用   | 休息 1 次 |
| 放松中    | 正在休息   |
| 蓬松状态   | 休息完成   |
| 小布丁趴睡  | 休息中    |
| 小布丁伸懒腰 | 休息完成   |

靠垫对象不需要太复杂，但要有明确“可点击”的感觉。

# 12. 开发实现建议

## 12.1 分层实现

建议不要将整页作为大图实现，而是组件化：

```text
角色组件
对象组件
贴纸组件
动作栏组件
浮层组件
统计组件
设置列表组件
```

## 12.2 贴纸数据结构

每次识别结果可转成：

```json
{
  "id": "record_001",
  "type": "food",
  "time": "08:30",
  "items": [
    {
      "name": "米饭",
      "sticker": "rice_bowl",
      "calories": 104
    }
  ],
  "totalCalories": 196
}
```

饮品：

```json
{
  "id": "drink_001",
  "type": "drink",
  "time": "15:20",
  "name": "奶茶",
  "volumeMl": 350,
  "tag": "偏甜",
  "sticker": "milk_tea"
}
```

休息：

```json
{
  "id": "rest_001",
  "type": "rest",
  "time": "16:40",
  "durationMin": 5
}
```

## 12.3 饭碗贴纸聚合

今日饭碗主视觉不要渲染所有识别项，先聚合代表项：

```text
按记录聚合
每条记录取 2–3 个代表贴纸
超过阈值显示 +N
点击进入明细抽屉看完整记录
```

一句话总结：现在 UI 物料的核心不是“多画几张可爱图”，而是要围绕 **小布丁、饭碗、水瓶、靠垫、贴纸、抽屉、动作栏** 建立一套可复用的组件资产。这样前端实现不会散，也能保证整个产品始终保持“轻陪伴 + 可记录 + 可统计”的统一心智。
