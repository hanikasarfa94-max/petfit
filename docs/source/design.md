# PetFit 页面设计说明 / 功能说明

## 0. 产品定位

PetFit 是一个轻陪伴式饮食、饮水、休息记录软件。

产品不是专业健康分析工具，也不是传统宠物养成游戏。核心心智是：

> 用户通过照顾小布丁，顺便照顾自己。

用户的行为会映射为三个核心对象的状态变化：

* 拍摄食物 / 饮料 → 饭碗 / 水瓶发生变化
* 快捷喝水 → 水瓶水位变化
* 休息一下 → 靠垫状态变化
* 长期记录 → 小布丁形成共同记忆

页面设计应避免传统健康 App 的强 BI 感。统计可以存在，但只作为对象页里的二级功能，由用户主动点击进入。

## 1.2 设计原则

1. 底部不做导航栏，只做动作栏。
2. 设置页放在右上角，不占底部导航。
3. 饭碗、水瓶、靠垫是对象入口，不是普通信息卡。
4. 日志和统计不独立成一级页面，而是收进对象页。
5. 统计页可以克制、准确、偏 BI，但必须由对象页主动进入。
6. 首页与对象默认页保持轻陪伴、低负担、低数据密度。
7. 拍摄入口统一，不区分拍食物还是拍饮料，由识别结果自动分流。

# 2. 小布丁中心页

## 2.1 页面目标

中心页是产品主入口，负责展示小布丁当前状态，并引导用户完成三个轻照顾动作：

* 休息一下
* 拍一下
* 喝口水

用户打开 App 后，不应感觉自己进入了健康看板，而应感觉进入了小布丁的日常空间。

## 2.2 页面结构

```text
小布丁中心页
├── 顶部区域
│   ├── 页面标题 / 小布丁状态文案
│   └── 右上角设置按钮
│
├── 中央场景区域
│   ├── 小布丁随机动画
│   ├── 饭碗小装饰
│   ├── 水瓶小装饰
│   └── 靠垫小装饰
│
└── 底部动作栏
    ├── 休息一下
    ├── 拍一下
    └── 喝口水
```

## 2.3 对象入口

中心页中展示三个小装饰对象：

### 饭碗

* 展示今天是否已有饮食记录。
* 可轻量显示饭碗内的食物贴纸。
* 点击进入饭碗页。

### 水瓶

* 展示今日水量大致状态。
* 如果用户拍摄过饮料，可在瓶内出现饮品贴纸。
* 点击进入水瓶页。

### 靠垫

* 展示今日休息状态。
* 点击进入靠垫页。

## 2.4 底部动作栏

底部动作栏固定为三枚按钮：

```text
休息一下 | 拍一下 | 喝口水
```

### 左侧：休息一下

* 小按钮。
* 点击弹出休息时长选择。
* 支持 3 分钟、5 分钟、10 分钟。
* 确认后进入休息中状态。
* 完成后记录一次休息，并更新靠垫状态。

### 中间：拍一下

* 最大主按钮。
* 统一拍摄入口，不区分食物或饮料。
* 可调用相机或相册。
* 拍摄后进入 AI 识别流程。
* 识别结果决定后续分流。

### 右侧：喝口水

* 小按钮。
* 只做快捷饮水记录。
* 点击后拉起底部浮层。
* 浮层内用滑动条或档位选择水量。
* 不进入水瓶页，不进入拍摄流程。
* 确认后水瓶水位上涨，并回到中心页。

# 3. 拍摄识别流程

## 3.1 拍摄入口

用户点击中心页底部「拍一下」后进入拍摄页。

拍摄页功能：

* 拍照
* 相册上传
* 返回
* 重新拍摄

不要求用户选择「拍食物」或「拍饮料」。

## 3.2 AI 识别逻辑

拍摄后调用视觉识别模型，识别图片中是否包含：

* 食物
* 饮料
* 食物 + 饮料
* 无关内容
* 模糊 / 不可识别内容

## 3.3 识别结果分流

### 只识别到食物

进入食物识别结果页。

展示：

```text
识别到的食物列表
每项估算热量
本次合计热量
可编辑名称 / 份量 / 热量
确认保存按钮
```

保存后：

```text
食物贴纸进入饭碗
默认优先展示即时反馈
用户可选择查看饭碗或回到中心页
```

### 只识别到饮料

进入饮料识别结果页。

展示：

```text
识别到的饮料
估算容量
是否偏甜
可编辑名称 / 容量
确认保存按钮
```

保存后：

```text
饮料贴纸进入水瓶
水瓶水位或饮品状态变化
用户可选择查看水瓶或回到中心页
```

### 同时识别到食物和饮料

默认进入食物识别结果页，饮料作为附加项展示。

页面提示：

```text
同时识别到饮料，保存后会一起记录到水瓶。
```

保存后：

```text
食物进入饭碗
饮料进入水瓶
默认主反馈给饭碗
```

原因：食物记录通常是更重的主任务，饮料作为同次记录附加保存。

### 无关 / 模糊图片

提示：

```text
没有识别到清晰的食物或饮料，可以重新拍一下。
```

提供：

```text
重新拍摄
返回
```

# 4. 饭碗页

## 4.1 页面目标

饭碗页用于展示用户今天拍摄 / 记录过的食物。

它不是传统饮食列表页，也不是统计页。默认状态应以「饭碗内的食物贴纸集合」为主，让用户看到：

> 今天给小布丁看过 / 放进饭碗的东西。

## 4.2 页面结构

```text
饭碗页
├── 顶部导航
│   ├── 返回按钮
│   ├── 标题：饭碗
│   └── 更多按钮
│
├── 饭碗主视觉
│   ├── 今日所有食物贴纸
│   ├── 今日已记录次数
│   └── 小布丁装饰
│
├── 日期滑动条
│   ├── 今天
│   ├── 前一天
│   ├── 后一天
│   └── 可左右滑动
│
├── 底部按钮
│   ├── 拍一下
│   └── 查看统计
│
└── 明细抽屉
    └── 点击饭碗后弹出
```

## 4.3 堆叠逻辑

饭碗中的贴纸不能无限无序堆叠。需要设计可控规则。

### 贴纸来源

每次成功识别食物后，系统根据识别结果生成 1 到 N 个食物贴纸。

示例：

```text
米饭 → 米饭贴纸
鸡肉 → 鸡腿 / 鸡胸肉贴纸
沙拉 → 蔬菜碗贴纸
水果 → 草莓 / 苹果 / 香蕉贴纸
面食 → 面碗贴纸
```

### 展示规则

当今日记录较少时，直接显示主要食物贴纸：

```text
1-4 次记录：完整展示主要贴纸
```

当今日记录较多时，进行分组和缩略：

```text
5-8 次记录：优先展示每次记录的代表贴纸
8 次以上：显示代表贴纸 + “+N”堆叠标记
```

### 代表贴纸优先级

每次记录最多选 2-3 个代表贴纸进入饭碗主视觉。

优先级建议：

```text
主食 / 主菜 > 饮品外的明显食物 > 水果 > 配菜 > 调料
```

避免一顿饭中所有细碎元素都进入饭碗，导致主视觉过满。

### 点击查看完整内容

饭碗主视觉只负责展示“今天吃到的东西集合”。
完整明细通过点击饭碗后弹出的抽屉查看。

## 4.4 饭碗明细抽屉

点击饭碗主视觉后，从底部弹出抽屉。

抽屉内容：

```text
今日饭碗明细
08:30  食物贴纸  156 kcal
11:40  食物贴纸  220 kcal
15:10  食物贴纸  360 kcal
19:10  食物贴纸  180 kcal
```

不要固定区分早餐 / 午餐 / 晚餐，因为用户可能一天记录很多次，也可能存在加餐、夜宵、补录。

展示字段：

```text
时间
代表食物贴纸
估算热量
详情入口
```

点击某条记录进入餐食详情页。

## 4.5 饭碗统计

点击「查看统计」进入饭碗统计页。

饭碗统计页可以偏克制、准确、数据化。

内容：

```text
时间筛选：7天 / 30天
平均热量
记录次数
卡路里趋势
相关记录入口
```

统计页不使用大量贴纸和 emoji，以保证可信度。

# 5. 水瓶页

## 5.1 页面目标

水瓶页展示今日饮水与饮品记录。

它包含两类记录来源：

1. 快捷喝水
2. 拍摄识别饮料

## 5.2 页面结构

```text
水瓶页
├── 顶部导航
├── 水瓶主视觉
│   ├── 今日水量
│   ├── 水位状态
│   ├── 饮料贴纸
│   └── 偏甜提示
├── 日期滑动条
├── 底部按钮
│   ├── 喝口水
│   └── 查看统计
└── 明细抽屉
```

## 5.3 喝口水快捷记录

点击中心页或水瓶页的「喝口水」后拉起底部浮层。

浮层内容：

```text
给小布丁加点水
水量滑动条：100ml - 500ml
快捷档位：100 / 200 / 300 / 500
确认按钮
```

确认后：

```text
水瓶水位上涨
写入饮水记录
回到上一级页面
```

## 5.4 拍摄饮料记录

通过「拍一下」进入拍摄流程。如果识别到饮料：

* 饮料贴纸进入水瓶。
* 如果识别为奶茶、可乐、果汁等高糖饮品，轻标记为「偏甜」。
* 语气不要警告化。

推荐文案：

```text
这杯有点甜，等会儿可以补一杯水。
```

避免：

```text
高糖风险，请减少摄入。
```

## 5.5 水瓶明细抽屉

点击水瓶后弹出今日明细：

```text
09:30  喝水  200ml
11:20  绿茶  300ml
15:40  奶茶  350ml  偏甜
18:10  喝水  200ml
```

## 5.6 水瓶统计

点击「查看统计」进入饮水统计页。

内容：

```text
近 7 天平均饮水量
达标天数
偏甜饮品次数
饮水趋势
相关饮品记录入口
```

# 7. 统计页通用模板

饭碗、水瓶、靠垫共用统计页模板，但根据对象切换标题、指标和颜色。

## 7.1 统计页结构

```text
统计页
├── 返回按钮
├── 标题
├── 时间范围
│   ├── 7天
│   └── 30天
├── 核心摘要
├── 趋势图
├── 对比信息
└── 查看相关记录
```

## 7.2 饭碗统计

```text
标题：饭碗统计
核心指标：平均热量 / 记录次数
趋势：卡路里趋势
相关记录：餐食记录
```

## 7.3 水瓶统计

```text
标题：水瓶统计
核心指标：平均饮水量 / 达标天数 / 偏甜饮品次数
趋势：饮水趋势
相关记录：饮水与饮品记录
```

## 7.4 靠垫统计

```text

核心指标：休息次数 / 平均休息时长
趋势：休息次数或时长趋势
相关记录：休息记录
```

## 7.5 统计页视觉原则

* 可使用柱状图、折线图、色阶。
* 不使用大量宠物贴纸。
* 不过度装饰。
* 因为用户是主动点「查看统计」进入，所以可以比主界面更理性。
* 但仍保持轻量，不做复杂专业健康分析。

# 9. 核心流转

## 9.1 主流程：拍一下

```text
小布丁中心页
→ 点击「拍一下」
→ 拍摄页
→ AI 识别
→ 识别结果确认
→ 保存
→ 根据识别结果反馈
```

分流：

```text
食物 → 饭碗反馈
饮料 → 水瓶反馈
食物 + 饮料 → 主反馈给饭碗，饮料同步进入水瓶
```

## 9.2 主流程：喝口水

```text
小布丁中心页
→ 点击「喝口水」
→ 底部浮层选择水量
→ 确认
→ 水瓶水位上涨
→ 写入水瓶记录
→ 回到中心页
```

## 9.3 主流程：休息一下

```text
小布丁中心页
→ 点击「休息一下」
→ 选择休息时长
→ 休息中
→ 完成反馈
→ 靠垫状态变化
→ 写入靠垫记录
```

## 9.4 对象查看流程

```text
中心页点击饭碗 → 饭碗页
中心页点击水瓶 → 水瓶页
中心页点击靠垫 → 靠垫页
```

对象页内：

```text
点击对象主视觉 → 展开今日明细抽屉
点击查看统计 → 进入该对象统计页
点击返回 → 回到上一级
```

# 10. 交互约束

## 10.1 不固定餐次

饭碗页不使用固定早餐 / 午餐 / 晚餐切换。

原因：

* 用户可能一天记录很多次。
* 用户可能有加餐、夜宵、补录。
* 固定餐次会降低灵活性。

饭碗明细以时间排序为主。

## 10.2 食物贴纸数量控制

饭碗主视觉必须控制贴纸堆叠。

规则：

```text
每次记录最多进入 2-3 个代表贴纸。
当天记录过多时，出现 +N 堆叠标记。
完整记录必须进入明细抽屉查看。
```

## 10.3 拍摄入口统一

不设置「拍食物」和「拍饮料」两个入口。

统一为：

```text
拍一下
```

由模型识别结果决定进入饭碗还是水瓶。

## 10.4 统计不污染主界面

主界面不直接展示复杂图表。

统计只在对象页点击「查看统计」后进入。

## 10.5 设置不占底部导航

设置统一放右上角。

# 12. 总结

最终方案采用：

```text
中心主页 + 三个对象页 + 统一拍摄入口 + 右上角设置
```

核心判断：

```text
底部不做导航，做行为动作。
饭碗 / 水瓶 / 靠垫不是信息卡，而是对象入口。
日志和统计不是独立页面，而是对象的展开能力。
拍摄入口不区分食物和饮料，由识别结果自动分流。
```

产品主心智：

> 用户不是在填写健康数据，而是在照顾小布丁；小布丁的饭碗、水瓶和靠垫会记录这些照顾痕迹。
