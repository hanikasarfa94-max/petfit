## 1. 识别结果映射表

现在有贴纸库，但还缺 **模型识别结果 → 贴纸 → 类型 → 是否进入饭碗/水瓶** 的映射规则。

需要补一个 `asset_mapping.json`：

```json
{
  "rice": {
    "zhName": "米饭",
    "category": "主食",
    "targetObject": "bowl",
    "sticker": "food_rice_bowl",
    "priority": 1
  },
  "milk_tea": {
    "zhName": "奶茶",
    "category": "饮品",
    "targetObject": "bottle",
    "sticker": "drink_milk_tea",
    "tag": "偏甜",
    "priority": 1
  }
}
```

否则模型返回“鸡胸肉 / 鸡肉 / 烤鸡 / chicken breast”时，前端不知道用哪个贴纸。

建议补充字段：

```text
识别名称
中文展示名
英文/别名
一级类别
进入对象：饭碗 / 水瓶
贴纸资源名
是否偏甜
是否默认代表贴纸
贴纸优先级
```

## 4. 贴纸堆叠布局规则物料

你已经提到堆叠规则，但还需要具体布局模板。

建议做 4 套饭碗布局：

```text
layout_bowl_1_3：1–3 个贴纸
layout_bowl_4_6：4–6 个贴纸
layout_bowl_7_10：7–10 个贴纸
layout_bowl_overflow：10+，显示代表贴纸 +N
```

每套需要明确：

```text
贴纸位置坐标
最大缩放比例
层级 z-index
是否允许旋转
是否允许遮挡
+N 标签位置
新贴纸飞入目标点
```

例如：

```json
{
  "layout": "bowl_4_6",
  "slots": [
    {"x": 0.28, "y": 0.32, "scale": 1.0, "rotate": -6},
    {"x": 0.52, "y": 0.28, "scale": 0.95, "rotate": 4},
    {"x": 0.70, "y": 0.42, "scale": 0.9, "rotate": -3}
  ]
}
```

这对 Codex 实现很关键。
