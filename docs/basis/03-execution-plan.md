# PetFit 执行计划

## 当前已经完成

- 已初始化 Git 仓库
- 已建立 `docs/`、`assets/`、`scripts/` 结构
- 已归档原始说明文档
- 已把核心 sprite sheet 改为语义化文件名

## 当前这一版交付的目标

- 形成后续工作的依据文档
- 建立图片资产自动切片/抠图脚本
- 生成第一版资产索引

## 下一阶段建议顺序

### 1. 资产标准化

- 复核自动切片结果
- 标注每个切片的语义名、类别、用途
- 拆出 P0 首屏必需资产

### 2. 设计到实现的映射

- 整理 `识别结果 -> 贴纸 -> 对象 -> 优先级`
- 整理 `对象状态 -> 所需图片/动画`
- 整理 `页面 -> 所需资产 -> 缺口`

### 3. 前端实现准备

- 定义 token
- 定义数据结构
- 定义页面组件树
- 明确哪些用静态图，哪些做真实组件

## Definition Of Done

当下面 4 项成立时，可以开始稳妥进入 UI 实现：

- 产品依据文档不再互相冲突
- 资产索引可查
- 切片资产可复用
- P0 页面所需资源已知且可定位
