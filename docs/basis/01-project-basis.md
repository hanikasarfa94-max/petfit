# PetFit 项目依据

## 1. 项目定位

`PetFit` 不是传统健康 BI App，也不是宠物养成游戏。

它的核心心智是：

> 用户通过照顾小布丁，顺便照顾自己。

用户行为会映射到三个对象：

- `饭碗`：食物记录与贴纸聚合
- `水瓶`：喝水与饮品记录
- `靠垫`：休息记录

## 3. 已确认的产品结构

应用 IA 采用：

```text
中心主页 + 三个对象页 + 统一拍摄入口 + 右上角设置
```

核心页面：

- 中心主页
- 拍摄页
- 识别结果确认页
- 保存反馈页
- 饭碗页
- 水瓶页
- 靠垫页
- 通用统计页
- 设置页

## 4. 关键产品原则

- 底部区域是动作栏，不是 Tab 导航。
- 设置入口固定在右上角。
- 拍摄入口统一为“拍一下”，不区分食物和饮料。
- 日志和统计不做一级页面，统一收纳到对象页内。
- 首页与对象默认页强调轻陪伴、低信息密度、低负担。
- 统计页允许更理性，但不做强专业医疗/健康分析。

## 5. 关键交互流

### 拍一下

```text
主页 -> 拍摄 -> AI 识别 -> 结果确认 -> 保存反馈
```

分流规则：

- `食物` -> 饭碗
- `饮料` -> 水瓶
- `食物 + 饮料` -> 主反馈给饭碗，饮料同步写入水瓶

### 喝口水

```text
主页/水瓶页 -> 底部浮层 -> 选择水量 -> 写入记录 -> 水位上升
```

### 休息一下

```text
主页/靠垫页 -> 底部浮层 -> 选择时长 -> 休息中 -> 写入记录
```

## 6. 视觉与体验原则

- 统一主角为 `小布丁仓鼠`
- 暖奶油色、浅木色、柔和高光、圆润治愈感
- 陪伴语气优先于管理语气
- 鼓励和提醒可以存在，但避免说教、警告化措辞
